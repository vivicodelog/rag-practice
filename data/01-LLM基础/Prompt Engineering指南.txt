Prompt Engineering 指南

一、什么是 Prompt Engineering
Prompt Engineering（提示词工程）是通过精心设计输入文本，引导语言模型产生期望输出的技术。它是当前使用 LLM 最核心的技能之一。

二、基本原则
1. 明确具体：告诉模型你想要什么，不要含糊
2. 提供上下文：给模型足够的信息来理解任务
3. 设定约束：告诉模型不能做什么（防幻觉、防跑题）
4. 分步骤引导：复杂任务拆成几步告诉模型

三、常用技巧

1. 角色设定
   "你是一个专业的技术文档助手，用简洁准确的语言回答。"

2. 格式控制
   "用以下格式回答：\n来源：【文件名】\n答案：..."

3. 防幻觉约束
   "如果你不知道答案，直接说不知道，不要编造。"

4. 示例引导（Few-shot）
   先给 1-2 个例子，再让模型按同样格式输出。

5. 思维链（Chain-of-Thought）
   让模型一步步思考，而不是直接给答案。

四、RAG 场景下的 Prompt 设计

系统提示词通常包含：
- 角色定义（你是谁）
- 工作流程（先搜什么，再做什么）
- 回答格式要求（来源标注等）
- 防幻觉规则（严格按文档回答）
- 边界处理（找不到怎么办）

五、常见误区
1. Prompt 太短：模型没有足够信息理解任务
2. 约束太少：模型容易自由发挥
3. 温度太高：输出不稳定，容易幻觉
4. 没有兜底策略：模型不知道"不知道时该怎么办"
