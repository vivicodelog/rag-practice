RAG 架构原理

一、什么是 RAG
RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与语言模型生成相结合的技术架构。核心思想是：在模型回答问题之前，先从知识库中检索相关文档，然后基于检索结果生成答案。

二、为什么需要 RAG
LLM 本身存在知识截止、幻觉、无法访问私有数据等问题。RAG 通过引入外部知识库，让模型能够：
1. 访问实时或私有数据
2. 减少幻觉，因为回答有据可查
3. 让用户可以更新知识库而不需要重新训练模型

三、RAG 标准流程
1. 索引阶段：文档 -> 文本切分 -> 向量化 -> 存入向量数据库
2. 检索阶段：用户问题 -> 向量化 -> 在向量库中搜索相似文档块
3. 生成阶段：问题 + 检索到的文档 -> 拼接 Prompt -> LLM 生成回答

四、进阶技术
1. 混合检索：向量检索 + 关键词检索互补，提高召回率
2. Rerank 重排序：用 CrossEncoder 对检索结果精排，提高精度
3. Query 改写：将用户问题改写为更适合检索的形式
4. 多轮检索：对于复杂问题，多次检索逐步求精

五、评估指标
- Hit@k：正确答案是否在前 k 条结果中
- MRR：正确答案排名的倒数均值
- Precision@k：前 k 条结果中相关文档的比例
- Recall@k：前 k 条结果覆盖了多少相关文档

六、RAG 与微调对比
RAG 适合需要频繁更新知识、降低幻觉的场景；微调适合让模型学习特定风格或专业术语。两者可以结合使用。
