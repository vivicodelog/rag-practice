向量数据库入门

一、什么是向量数据库
向量数据库是一种专门存储和检索向量（高维数值数组）的数据库。在 RAG 系统中，它将文档的语义表示为向量，并通过相似度搜索找到与用户问题语义最匹配的文档。

二、常见向量数据库对比

ChromaDB（本项目使用）
- 开源、轻量、易于上手
- 支持内存模式和持久化
- 适合小到中型项目

FAISS（Meta 开源）
- 纯向量搜索库，比 Chroma 更快
- 不提供数据库功能（需要自己管理 ID 映射）
- 适合大规模检索

Milvus
- 分布式向量数据库
- 支持百亿级别向量
- 适合生产环境大规模部署

Pinecone / Weaviate
- 云服务，无需自己运维
- 成本较高

三、核心概念
1. 向量维度：每个向量的长度，bge-small-zh 是 512 维
2. 距离度量：余弦相似度、欧氏距离、内积
3. 索引类型：暴力搜索（准确但慢）、近似搜索（快但略有精度损失）
4. 嵌入模型：将文本转为向量的模型，如 bge-small-zh-v1.5

四、检索流程
文本 -> 嵌入模型 -> 向量 -> 存入向量库
问题 -> 相同的嵌入模型 -> 查询向量 -> 最近邻搜索 -> 返回文档

五、关键考虑
1. 嵌入模型选择：中文场景推荐 bge 系列
2. 向量维度：维度越高越精确但越慢
3. 索引构建时机：增删文档后需要重建索引
4. 混合检索：向量搜索与关键词搜索互补，效果更好
